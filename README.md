# Mirtek ESPHome Gateway

ESPHome-прошивка для беспроводных счётчиков электроэнергии **Миртек (Star 104 / Star 304)** через радиомодуль **CC1101** (433 МГц).

Интеграция с **Home Assistant** по API и **MQTT**, встроенный **Web-сервер**, вывод на **OLED SSD1306**.

---

## Возможности

| Функция | Описание |
|---------|---------|
| 📡 CC1101 | Приём/передача на 433 МГц, полный протокол Star v1.20 |
| ⚡ 3 тарифа | Активная + реактивная энергия по T1/T2/T3 и суммарно |
| 📊 Мгновенные | U, I, P, Q, S, cos φ по каждой фазе + частота |
| 🔌 Реле | Управление реле отключения из HA / MQTT / кнопки |
| 🛡 Пломбы | Статус всех электронных пломб и защит |
| 🏷 Паспорт | Серийный №, версия ПО, тип счётчика, абонент |
| 🔋 Батарея | Ресурс батарейки счётчика (остаток в %) |
| 🖥 OLED | 4 страницы с автопролистыванием (энергия / UI / мощность / инфо) |
| 🌐 Web UI | Встроенный веб-сервер на порту 80 |
| 📨 MQTT | Все показания + команды через MQTT |
| 🏠 HA | Автообнаружение через ESPHome API + MQTT Discovery |
| 🔄 OTA | Обновление прошивки по воздуху |

## Схема подключения

### CC1101 → ESP32

```
CC1101     ESP32
------     -----
VCC   →   3.3V   ⚠️ Только 3.3В, не 5В!
GND   →   GND
SCK   →   GPIO18
MOSI  →   GPIO23
MISO  →   GPIO19
CSN   →   GPIO5
GDO0  →   GPIO22  ← прерывание приёма
GDO2  →   не нужен
```

### SSD1306 OLED → ESP32 (опционально)

```
OLED       ESP32
----       -----
VCC   →   3.3V
GND   →   GND
SDA   →   GPIO21
SCL   →   GPIO22  ← если нет конфликта с GDO0, иначе сменить
```

> **Конфликт пинов:** если GDO0 и SCL оба на GPIO22 — перенесите GDO0 на GPIO4 и обновите `mirtek.yaml`.

## Добавление в Home Assistant

После первого запуска ESP32 появится в **Настройки → Интеграции → ESPHome** автоматически через mDNS.

При запросе введите `ha_api_key` из `secrets.yaml`.

Все сущности (сенсоры, кнопки, бинарные сенсоры) создаются автоматически.

## Параметры конфигурации

Все настройки в секции `substitutions` файла `mirtek.yaml`:

| Параметр | По умолчанию | Описание |
|----------|-------------|---------|
| `meter_address` | `****` | Адрес счётчика (1–65000) |
| `poll_interval` | `300s` | Интервал опроса (минимум 60s) |
| `mqtt_prefix` | `mirtek` | Префикс MQTT топиков |
| `display_flip_s` | `6` | Секунд на каждой странице OLED |

## MQTT топики

Базовый топик: `mirtek/<meter_address>/`

| Топик | Тип | Описание |
|-------|-----|---------|
| `.../status` | text | `online` / `offline` |
| `.../sensor/суммарное_потребление/state` | float | кВт·ч |
| `.../sensor/тариф_1_день/state` | float | кВт·ч |
| `.../sensor/тариф_2_ночь/state` | float | кВт·ч |
| `.../sensor/тариф_3_полупик/state` | float | кВт·ч |
| `.../sensor/активная_мощность/state` | float | кВт |
| `.../sensor/напряжение_фаза_1/state` | float | В |
| `.../sensor/ток_фаза_1/state` | float | А |
| `.../sensor/частота_сети/state` | float | Гц |
| `.../cmd` | command | `1`=опрос, `8`=реле ON, `9`=реле OFF |

## Страницы OLED-дисплея

| # | Страница | Содержимое |
|---|----------|-----------|
| 1 | Энергия | Суммарно + T1/T2/T3 (кВт·ч, кВАр·ч) |
| 2 | U / I | Напряжение и ток по фазам, частота, cos φ |
| 3 | Мощность | P суммарно, Pa/Pb/Pc, температура |
| 4 | Информация | Дата/время счётчика, реле, пломба, серийный № |

Ручное переключение: кнопка **"Следующая страница дисплея"** в Home Assistant.

## Управление реле

Три способа:

**Home Assistant** — кнопки «Включить реле» / «Выключить реле»

**MQTT**:
```bash
mosquitto_pub -h 192.168.1.100 -t "mirtek/*****/cmd" -m "8"   # включить
mosquitto_pub -h 192.168.1.100 -t "mirtek/*****/cmd" -m "9"   # выключить
```

**HA Service call**:
```yaml
service: esphome.mirtek_gw_relay_on
```

## Диагностика

### Логи в реальном времени
```bash
esphome logs mirtek.yaml
```

### Типичный успешный вывод
```
[I][mirtek] Init CC1101 cs=5 gdo0=22 addr=*****
[I][mirtek] CC1101 OK, RX mode
[I][mirtek] === Poll #1 addr=***** ===
[I][mirtek] Type=3ф трансф.акт-реакт FW=05.12 Relay=Вкл Seal=OK
[I][mirtek] kWh SUM=15234.56 T1=10123.45 T2=4890.11 T3=221.00
[I][mirtek] U=228.5/229.1/228.8 I=1.234/1.189/1.201 P=0.843kW f=50.02 cos=0.987
[I][mirtek] Pa=281 Pb=272 Pc=290 T=28°C
[I][mirtek] === Done ===
```

### Уровень логирования

Для диагностики в `mirtek.yaml`:
```yaml
logger:
  level: DEBUG    # или VERBOSE для максимальной детализации
```

### CC1101 не найден
- Проверьте пайку SPI-пинов
- Убедитесь что VCC = 3.3В
- Проверьте `logger: level: DEBUG` — должна быть строка `CC1101 OK`

### Нет данных от счётчика
- Убедитесь что адрес счётчика верный
- Попробуйте поднести антенну ближе
- Включите `VERBOSE` и следите за `timeout cmd=`

## Несколько счётчиков

Скопируйте `mirtek.yaml` в `mirtek2.yaml` и измените:
```yaml
substitutions:
  device_name:    mirtek-gw2
  meter_address:  "12345"   # адрес второго счётчика
```

Прошейте на второй ESP32:
```bash
esphome run mirtek2.yaml
```

## Протокол

Реализован проприетарный радиопротокол Star v1.20 (Миртек):

- **Частота:** 433 МГц
- **Модуляция:** FSK через CC1101
- **CRC8:** полином `0xA9`, начальное значение `0x00`
- **Байтстаффинг:** `0x55 → {0x73, 0x11}`, `0x73 → {0x73, 0x22}`
- **Порядок байт:** Little-Endian
- **Знаковые значения:** старший бит — знак (не дополнительный код)

Реализованные команды:

| Код | Команда | Описание |
|-----|---------|---------|
| `0x01` | Ping/Ident | Тип счётчика, версия ПО, реле, пломбы |
| `0x05` | ReadEnergy | Энергия: акт./реакт., T1/T2/T3 + сумма |
| `0x07` | ReadAbonInfo | Данные абонента (лицевой счёт, ФИО) |
| `0x0A` | ReadFactoryStr | Заводской №, дата пр-ва, наименование |
| `0x1C` | ReadDateTime | Дата и время на счётчике |
| `0x1E` | ReadResBat | Ресурс батарейки |
| `0x2B/00` | ReadInstantValue | U, I, P, Q, f, cos суммарные |
| `0x2B/10` | ReadInstantValue | P, Q, S, cos по фазам + температура |
| `0x30` | GetInfo | Время работы, версия платы |
| `0x3A` | OperateOverload | Управление реле отключения |
